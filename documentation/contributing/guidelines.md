# Contribution Guidelines

We are excited that you are interested in contributing to Nerion V2! To ensure a smooth and collaborative process, please follow these guidelines.

## Getting Started

1.  Fork the repository and create a new branch for your feature or bug fix.
2.  Set up your development environment by following the instructions in the [Development Setup](./development-setup.md) guide.
3.  Make your changes and write tests to cover your new code.
4.  Run the tests to ensure that your changes do not break any existing functionality.
5.  Format your code using a code formatter to ensure a consistent code style.
6.  Submit a pull request with a clear and descriptive title and a detailed description of your changes.

## Pull Request Process

1.  Your pull request will be reviewed by one of the project maintainers.
2.  The maintainer may ask you to make changes to your code or to provide more information.
3.  Once your pull request has been approved, it will be merged into the main branch.

## Code Style

We use a code formatter to ensure a consistent code style throughout the project. Please make sure to format your code before submitting a pull request.

## Issue Tracker

If you find a bug or have a feature request, please open an issue on our issue tracker. Please provide as much detail as possible, including steps to reproduce the bug or a clear description of the feature you would like to see.
